# Task: Autonomous forensic wiki reports for high-risk MPs

## Objective
Generate forensic wiki reports for high-risk MPs identified by the PostgreSQL query.

## Data Sources
- `leaderboard.json` (API data, if available)
- PostgreSQL query results
- Web search results

## Entity Resolution Strategy
- Use the MP's ID, display name, and current party from the PostgreSQL query.
- Fetch forensic data from the API if available, otherwise use the SQL summary.

## Cross-Dataset Linking Approach
- Link the MP's ID to the API and web search results.
- Ensure all data points are correctly attributed to the MP.

## Evidence Chain Construction
- For each MP, build an evidence chain that includes:
  - MP's ID, display name, and current party from the PostgreSQL query.
  - Forensic data from the API or SQL summary.
  - Web search results for additional evidence.

## Expected Deliverables and Output Format
- For each MP, generate a Markdown file in the `dashboard/public/wikis/` directory with the following sections:
  - `## Summary` (cited facts only)
  - `## Forensic Engine Findings` (table of forensic data)
  - `## Web Evidence` (links or "No corroborating web evidence found")
  - `## Conclusion` (Low|Medium|High|Critical)
- Create an `index.json` file in the `dashboard/public/wikis/` directory containing a JSON array of `{ "mp_id", "display_name", "risk_level", "wiki_path": "/wikis/<mp_id>.md" }` for MPs with a written `.md`.

## Risks and Limitations
- The API might not be available or might not return the necessary data.
- Web search results might be limited or not available.
- The PostgreSQL query might not return the expected results.

## Next Steps
1. **Forensic Data**: For each MP, fetch forensic data from the API if available, otherwise use the SQL summary.
2. **Web Search**: Perform a web search for each MP to gather additional evidence.
3. **Markdown Sections**: Write the summary, forensic engine findings, web evidence, and conclusion sections for each MP.
4. **Write File**: Save the Markdown content to the appropriate file in the `dashboard/public/wikis/` directory.
5. **Index File**: Generate the `index.json` file with the list of MPs and their risk levels.
